## 🔑 API Key Options

To use Zernio with Twitter/X, you have two options for API credentials: use the communal API key provided by Dalek, or set up your own personal API key.

<Aside type="caution">
**Twitter/X has discontinued their free tier API.** All API access now operates on a pay-as-you-go model. This affects both the communal key and any personal keys you set up yourself.
</Aside>

    ## Setting Up Your Twitter/X API Credentials

    <Steps>
      <Step title="Go to Twitter Developer Portal">
        Navigate to the [Twitter Developer Portal](https://developer.twitter.com/) and sign in with your Twitter/X account.
      </Step>

      <Step title="Create a Developer Account">
        If you don't already have one, you'll need to apply for a developer account. Fill out the use case form when prompted.

        <Aside type="tip">
        💡 **If asked to describe your Twitter API use cases (250 characters), copy this text:**

        > Our application enables users to schedule and publish text and media content to their own Twitter accounts using their personal API credentials. We access user profile data, publish tweets with media attachments, retrieve engagement metrics, and manage authentication.
        </Aside>
      </Step>

      <Step title="Create a New Project and App">
        From your developer dashboard, create a new **Project**, then create an **App** inside it. Give it a name you'll recognize (e.g., "Zernio Scheduler").
      </Step>

      <Step title="Set Spend Limits">
        Before you do anything else, go to your **Billing & Usage** settings and configure a monthly spend cap.

        <Aside type="caution">
        **Set a spend limit — seriously.** Twitter/X operates on pay-as-you-go pricing with no free tier. Without a cap, unexpected usage spikes can result in charges you didn't plan for. Set a limit you're comfortable with before connecting your key to any app.
        </Aside>
      </Step>

      <Step title="Configure App Permissions">
        In your App settings, set the following permissions:
        - **Read and Write** (for posting tweets)
        - **Read Users** (for profile access)

        Make sure you're on at least the **Basic** tier — this is required for write access (posting tweets).
      </Step>

      <Step title="Navigate to Keys and Tokens">
        Go to the **Keys and Tokens** tab in your App settings.
      </Step>

      <Step title="Generate and Copy Your Credentials">
        Generate and copy the following:
        - **API Key** (Client ID)
        - **API Key Secret** (Client Secret)

        Store these somewhere safe — you won't be able to view the secret again after leaving the page.
      </Step>

      <Step title="Add the Redirect URI">
        Under **User Authentication Settings**, add this redirect URI to your app:

        `https://Zernio.com/api/v1/connect/twitter/callback`
      </Step>

      <Step title="Connect with Dalek">
        Reach out to Dalek through [OshiHub](https://hub.oshi.social/) and provide your credentials. He'll help get your account configured in Zernio.
      </Step>

      <Step title="Start Using Zernio">
        You're all set! Start scheduling your posts.
      </Step>
    </Steps>
